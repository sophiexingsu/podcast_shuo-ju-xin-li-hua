# Production Workflow

This is the default episode pipeline. Keep it boring and repeatable.

## 1. Intake

- Add the idea to `context/topic-backlog.md`.
- Decide why the episode matters now.
- Name the intended listener and the question the episode answers.

## 2. Research

- Create a folder from `episodes/_template/`.
- Fill in `brief.md`.
- Save source links and notes in `research.md`.
- Mark unverified claims clearly until checked.

## 3. Outline

- Draft the episode path in `outline.md`.
- Confirm intro, key beats, transitions, and closing ask.
- Avoid over-scripting unless the format calls for it.

## 4. Record

- Use `recording-notes.md` before and during the session.
- Capture guest names, pronunciation notes, and any retakes.
- Keep raw recordings outside Git unless the team deliberately changes that policy.

## 5. Post-Production

- Edit audio.
- Generate or attach transcript.
- Write title, description, show notes, chapters, links, and social copy.
- Run the QA checklist before publishing.

## 6. Publish and Promote

- Publish using the host platform checklist.
- Post clips or copy to selected channels.
- Save final public links in the episode folder.

## 7. Review

- Add performance notes after launch.
- Capture what to repeat, change, or stop.
- Move useful learnings into `context/audience-notes.md`.
