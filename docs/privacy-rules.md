# Privacy Rules

Default to private. The repo can contain planning context, templates, decisions, episode briefs, and non-sensitive research notes.

## Do Commit

- Show bible and audience notes.
- Episode briefs, outlines, show notes, and public links.
- Research links and source summaries.
- Publishing checklists and retrospective notes.
- Non-sensitive asset inventories.

## Do Not Commit

- API keys, passwords, session tokens, or platform credentials.
- Raw unreleased audio or video recordings.
- Guest phone numbers, private emails, addresses, or contracts.
- Payment information, ad account credentials, or sponsor deal terms unless explicitly approved.
- Local `.env` files or machine-specific settings.

## Raw Media

Store raw media in a separate cloud drive or local folder. If the team later wants Git-backed large media, use Git LFS deliberately and update this doc first.

## Guest Consent

Before publishing guest material, confirm the guest understood the recording and release expectations. Save only the level of consent detail the team is comfortable storing in this private repo.

## Public Exports

Final public show notes, transcripts, and approved clips can be copied into this repo if the team wants a complete archive.
